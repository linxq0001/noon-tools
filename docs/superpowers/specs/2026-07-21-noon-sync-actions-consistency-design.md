# Noon 同步操作组一致性设计

## 目标

统一 Noon 工作台顶部同步卡和空数据状态中的“API 快速同步 / 导出同步”操作，使两处按钮顺序、尺寸、间距、禁用状态和说明一致，同时保留各自的布局语境。

## 设计

- 提取单一 `CatalogSyncActions` 组件，接收当前模式、同步状态、店铺可用状态和启动回调。
- 操作组固定按“API 快速同步 → 导出同步”排列，两枚按钮均使用青色主按钮样式，桌面端横向紧邻并使用统一间距，不再由父容器 `space-between` 拉散。
- 顶部同步卡分为状态区和操作区：状态文案在左，操作组靠右。
- 空数据状态复用同一操作组并整体居中，不再单独渲染两枚按钮。
- FBN/FBP 模式继续禁用“导出同步”；操作组通过现有可见说明和 `aria-describedby` 表达原因。
- 小于等于 760px 时，顶部同步卡纵向排列；操作组允许换行并保持按钮相邻，不制造横向溢出。

## 状态与行为

- 不改变同步 API、任务轮询、错误处理或快照读取逻辑。
- 两处入口共享同一 `disabled` 计算，避免状态漂移。
- 同步运行或未选择店铺时，两枚按钮均按现有规则禁用。

## 验证

- 静态契约测试确认页面只定义一套同步操作组件且使用两次。
- 测试确认按钮顺序、共享禁用条件、FBN/FBP 提示关联和移动端回退。
- 运行 TypeScript 检查和 Next.js 生产构建。
- 视觉复核桌面端顶部靠右/空状态居中，以及移动端无溢出。

## 不做事项

- 不调整颜色、字体、业务文案或同步流程。
- 不删除任一入口。
- 不引入新依赖。
