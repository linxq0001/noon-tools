# Domain docs

This repo uses a single-context layout.

Consumer rules:

- Read `CONTEXT.md` at the repo root when domain vocabulary or project background matters.
- Read ADRs from `docs/adr/` when architectural decisions may affect the task.
- If `CONTEXT.md` or `docs/adr/` do not exist yet, note that they are missing and proceed from local code context.
- Do not invent domain terminology; prefer names already used in the codebase and docs.
