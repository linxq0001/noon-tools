# 勘察全栈任务

先读取事实，不写代码。检查项目指令、需求、设计、`package.json`、锁文件、路由、服务层、Prisma Schema、迁移、测试、部署与环境变量示例。识别新旧项目、实际技术栈、受影响层、可复用模式、资料冲突和生产风险。

输出：任务范围、不做事项、相关文件、外部 Skills 可用情况、风险等级、验证方式和下一阶段。发现资料冲突或生产操作时停止，等待用户决定或具体授权。
