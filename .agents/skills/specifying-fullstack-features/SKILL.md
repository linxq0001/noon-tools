# 定义全栈需求

优先调用 `brainstorming`，不可用时使用本契约。定义用户、角色、权限、页面状态、API 行为、数据模型、异常、重试、并发、幂等和范围外事项。每个关键行为写出触发条件、可见结果和可验证标准。用户确认前不得实现。

输出精简 Spec 或验收清单，避免“体验良好”等不可验证描述。
