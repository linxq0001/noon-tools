# 设计前端界面

所有 UI 优先调用 `frontend-design`。定义具体对象、用户、页面唯一目标、4～6 个颜色 token、字体角色、布局、标志性元素、加载/空/错误状态和响应式规则。有设计来源时严格还原，无设计来源时提出 2～3 个方向并等待确认。

Landing、Portfolio、Marketing、Editorial 和适用重设计叠加 `design-taste-frontend`；后台、密集数据表、多步骤表单和复杂产品工作台禁止调用。输出可实现且可视觉验证的设计契约。
